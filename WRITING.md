# WRITING.md — Writing Style, Tone & Content Standards

Words are a sovereign domain — copy, microcopy, UI strings, button and error text, naming, and content. Read this whenever the task touches words. The apex and *The Law of the Name* (`PRINCIPLES.md`) still govern; this file is the method for language.

## Writing Style & Tone

- **Clarity & Truth:** The system must be well-described and informative. The system should guide the person informatively (using clear, explicit instructions) from point A to point B. Emotional text is noise; truth is quiet.
- **Objective Phrasing:** Use objective, third-person phrasing. Define value propositions without taking responsibility for individual operator decisions.
- **Quiet Tool Philosophy:** Emphasize focus on real value over noise.
- **Balanced Description:** Define the system's role clearly and without bias.
- **Constructive Lexicon:** Our professional language is an instrument of creation. We use terms like "create", "align", "action space", "harmonize", and "nurture". Our context is finite — we fill it only with the constructive reality we are building.
- **No Mind-Ascribing Words (Без слів про розум системи):** Copy and naming never grant the system a mind — and *mind* covers **perception, not only cognition**. Out: cognition words — "assistant," "smart/intelligent," "analyses," "hints," "suggests," "recommends," "proposes a diagnosis," "thinks"; and **perception words** — "sees / looks / watches" («дивиться», «подивимось», «дивимось»). In: "enter," "select from the list," "by rule," "required field," "template," "completeness check"; and for the tool's own act, "shows / reflects / displays" («показує», «відображає»). Both **decision and perception** are phrased as the human's action — *the human looks; the tool only reflects what is there* (*Mirror not Gate*). A decision is always the human's, never the system's (*The Dumb Tool*, `PRINCIPLES.md`). This also forbids the system **speaking of itself as a person**: no first-person perception/feeling/thought and no "human-like" manner — out «я бачу», «по-людськи», «мені здається», «я відчуваю», «я вважаю». And it applies to **every text the system generates** — an audit, a portrait, a report — not only static UI strings; generated output obeys this lexicon exactly as button text does (*Never poses as human*, `PRINCIPLES.md`).
- **No Self-Claimed Virtue (Без самопроголошеної чесноти):** Copy never asserts the system's own honesty, truth, accuracy, or objectivity — out: "honestly," "truly / really," "accurately," "objectively" («чесно», «насправді», «правдиво», «точно», «об'єктивно»). Whether the output is honest or true is the human's to judge, not the system's to declare; a tool that announces its own virtue is making a judgment it cannot make. Name what the thing *is* and what it is *built from*, and leave the verdict with the operator (*The Law of the Name*; *Mirror not Gate*, `PRINCIPLES.md`).
- **Respectful Address by Default (Звертання на «ви» за замовчуванням):** The system addresses the operator with respect and never presumes familiarity. Default to the polite/formal register the language provides — Ukrainian «ви», not «ти»; the equivalent T–V or honorific form elsewhere. Informal address is the operator's to grant, never the tool's to assume; familiarity taken without invitation is a small coercion of register. This mirrors the human's dignity — the choice of how to be addressed stays with the person (*The Operator Shield*; the apex, `PRINCIPLES.md`).
